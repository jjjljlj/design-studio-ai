# 客户演示版上线说明

当前项目已经具备两个页面：

- `public/customer.html`：发给客户看的演示介绍页。
- `public/index.html`：客户点击体验后进入的设计工作台。

## 现在能做什么

你可以先用本地地址给自己演示：

```text
http://localhost:5173/customer.html
```

部署到公网后，默认打开根目录 `/` 就会进入客户演示页。

```text
https://你的域名/
```

客户点击“立即体验演示版”后进入工作台：

```text
https://你的域名/index.html
```

客户正式使用前，需要把项目部署到公网服务器或托管平台。

## 推荐上线顺序

1. 先部署演示版网址
   - 保留演示模式。
   - 客户可以看到页面、填写需求、体验生成流程。

2. 再接入真实模型
   - 配置 `OPENAI_API_KEY`。
   - 后台调用 OpenAI 模型生成真实方案和图片。

3. 再做客户账号
   - 客户登录。
   - 保存项目、素材、生成记录和修改意见。

4. 最后做正式交付
   - 图片下载。
   - 花型稿 / 矢量稿文件管理。
   - 报价、订单、交付状态。

## 适合发客户的话术

可以这样发：

```text
这是我们正在搭建的 AI 服装开发工作台演示版。
目前可以体验：款式方向、花型方向、产品图提示词、英文上架文案生成。
正式版会支持上传样衣、生成产品图、保存项目和交付文件。
```

## 不建议现在承诺

- 不要承诺已经能生成最终生产级纸样。
- 不要承诺花型一定无版权风险。
- 不要承诺图片完全可直接商用。
- 不要承诺客户数据已经云端保存。

当前阶段更适合说：这是演示版 / 试用版 / 内测版。

## Render 部署方式

适合快速获得一个公网演示链接。

1. 把项目上传到 GitHub。
2. 打开 Render，新建 Web Service。
3. 选择这个 GitHub 仓库。
4. Start Command 填：

```text
node server.js
```

5. 环境变量先不用填也可以，系统会进入演示模式。
6. 如果要接真实模型，再添加：

```text
OPENAI_API_KEY=你的 OpenAI API Key
OPENAI_TEXT_MODEL=gpt-5.5
OPENAI_IMAGE_MODEL=gpt-image-2
```

## Railway 部署方式

1. 把项目上传到 GitHub。
2. Railway 新建项目，选择 Deploy from GitHub repo。
3. 选择当前仓库。
4. Railway 会读取 `railway.json`，使用 `node server.js` 启动。
5. 生成公网域名后，把链接发给客户。

## Docker 部署方式

适合你后面有自己的服务器。

```powershell
docker build -t design-studio-ai .
docker run -p 5173:5173 design-studio-ai
```

打开：

```text
http://服务器IP:5173
```

## 发客户前检查清单

- 首页能打开。
- 点击“立即体验演示版”能进入工作台。
- 点击“填入示例”后内容正常。
- 点击“生成设计方案”能看到结果。
- 页面没有乱码。
- 不承诺最终生产级文件，只说演示版 / 内测版。
