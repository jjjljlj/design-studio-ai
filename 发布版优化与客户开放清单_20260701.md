# Design Studio AI 发布版优化与客户开放清单

## 当前已完成

- 客户首页已改为预约试用入口，主按钮指向“预约试用”。
- 试用申请表单已增加必填校验：姓名/昵称、邮箱或手机号、核心问题、数据用途同意。
- 服务端 `/api/feedback` 已增加同样的校验，避免空线索进入后台。
- 管理页已增加“导出试用线索 CSV”按钮。
- 后台导出接口已增加：`/api/admin/feedback.csv`，需要 `ADMIN_PIN`。
- 客户注册/试用信息保存位置已写入 `PERSISTENT_STORAGE.md`。

## 客户注册信息保存位置

正式配置 Supabase 时：

- 数据表：`public.design_records`
- 记录类型：`record_type = 'feedback'`
- 客户信息：保存在 `payload` JSON 字段
- 后台查看：`/admin.html`
- CSV 导出：`/api/admin/feedback.csv`

未配置 Supabase 时：

- 本地文件：`data/feedback.jsonl`
- 风险：Render 重启或重新部署后可能丢失，不适合正式开放。

## 对外开放前必须确认

1. OpenAI API 账单硬额度
   - 当前图片接口已接通，但账户返回 `Billing hard limit has been reached`。
   - 必须先解除额度限制，否则客户无法在线生成产品图。

2. Supabase 持久化
   - 正式开放前必须确认 `/admin.html` 的“数据保存”显示为“云数据库”。
   - 如果显示“临时文件”，先不要大规模引流。

3. 管理密码
   - Render 环境变量必须设置 `ADMIN_PIN`。
   - 不要把 `ADMIN_PIN` 发给客户。

4. 服务边界文案
   - 页面已说明真实上架、价格、库存、广告、平台账号操作需要人工确认。
   - 后续如接入真实平台账号，必须再加二次确认。

5. 客户隐私与授权
   - 当前是轻量数据用途同意。
   - 如果正式投放广告获客，建议补充完整隐私政策页面，包括数据用途、保存期限、联系方式和删除请求方式。

## 下一轮建议

- 加一个独立的 `/privacy.html` 隐私说明页。
- 给客户试用申请加来源字段，例如 TikTok、微信、独立站、朋友介绍。
- 后台增加线索状态：新线索、已联系、已报价、已成交、暂缓。
- 增加客户案例页，但只使用我们自己有授权的素材。
- 图片额度恢复后，只回测 1 条 `Blush Petal Camisole Set` 产品图，确认线上链路稳定后再开放给客户。
