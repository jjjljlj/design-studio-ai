# 稳定素材库保存设置

当前应用会保存三类数据：

- Lina 生成的款式、花型、图片提示词和素材库记录
- 工作台保存的客户确认项目
- 客户首页提交的试用需求

Render 免费服务的本地文件可能会在重启或重新部署后清空。为了长期保存数据，建议接入 Supabase 数据库。

## 1. 在 Supabase 创建数据表

进入 Supabase 项目后，打开 SQL Editor，执行：

```sql
create table if not exists public.design_records (
  id bigint generated by default as identity primary key,
  record_type text not null,
  record_id text not null unique,
  created_at timestamptz not null,
  updated_at timestamptz not null,
  payload jsonb not null
);

create index if not exists design_records_type_created_idx
  on public.design_records (record_type, created_at desc);

create index if not exists design_records_type_updated_idx
  on public.design_records (record_type, updated_at desc);
```

## 2. 在 Render 填环境变量

进入 Render 服务的 Environment 页面，添加：

```text
SUPABASE_URL=https://你的项目.supabase.co
SUPABASE_SERVICE_ROLE_KEY=你的 service_role key
SUPABASE_TABLE=design_records
```

保存后选择 `Save, rebuild, and deploy`。

## 3. 检查是否生效

打开管理页 `/admin.html`，输入管理密码后查看“数据保存”：

- 显示“云数据库”：已经接入稳定保存。
- 显示“临时文件”：还没有配置 Supabase，Render 重启后仍可能丢数据。

## 数据保护提醒

- `SUPABASE_SERVICE_ROLE_KEY` 只能填写在 Render 环境变量里，不要发给客户，不要提交到 GitHub。
- 客户确认页只读取指定项目，不暴露素材库列表。
- 管理页、Lina 员工页、生成接口仍由 `ADMIN_PIN` 保护。
