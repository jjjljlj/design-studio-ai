# 一键部署指南（给你直接用）

以下操作把当前项目从本地直接变成可让客户访问的线上页面。

## 第一步：先配置本地仓库

1. 打开终端，进入项目目录（`E:\数码描稿调色`）
2. 确认 `.env` 不在 Git 提交里（`.gitignore` 已有）
3. 运行：

```powershell
git status
```

只要看到你改动的文件正常即可。

## 第二步：运行一键脚本

双击 `deploy-github-render.bat`，按提示完成：

- 是否提交（默认提交说明）
- 是否配置 `origin`
- 是否推送到 GitHub

脚本会在控制台显示每一步结果。

## 第三步：Render 上线

1. 进入 Render 控制台
2. 点击 **New → Web Service**
3. 选择你刚创建的 GitHub 仓库
4. 配置项：

```text
Environment: Node
Build Command: 留空
Start Command: node server.js
```

5. 添加环境变量（`Environment Variables`）：

```text
OPENAI_API_KEY=<你的真实 key>
OPENAI_TEXT_MODEL=gpt-5.5
OPENAI_IMAGE_MODEL=gpt-image-2
PORT=5173
```

## 第四步：上线自检

打开 Render 给你的域名，检查：

- `https://你的域名/`（客户演示页）
- `https://你的域名/index.html`（工作台）
- 在演示页提交一次“试用申请”，确认成功提示

## 第五步：对外交付给客户

你可以直接发：

- 客户演示页链接：`https://你的域名/`
- 说明：默认是可交付演示，后续需接入图片存储和客户账号体系再做正式版交付。
